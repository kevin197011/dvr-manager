## ADDED Requirements

### Requirement: 审计日志记录

系统 SHALL 对以下可审计操作写入审计日志：登录成功、登录失败、录像查询请求、管理端配置变更。每条记录 SHALL 包含时间、动作类型、用户名（若有）、客户端 IP、资源标识（若有）、简要详情及状态（成功/失败）。

#### Scenario: 登录成功写入审计

- **WHEN** 用户登录成功
- **THEN** 系统写入一条审计记录，动作为登录成功，包含该用户名与客户端 IP

#### Scenario: 登录失败写入审计

- **WHEN** 用户登录失败（如密码错误）
- **THEN** 系统写入一条审计记录，动作为登录失败，包含尝试使用的用户名与客户端 IP

#### Scenario: 录像查询写入审计

- **WHEN** 发生录像查询请求（单个或批量）
- **THEN** 系统写入审计记录，包含动作类型、请求者身份（若已登录）、客户端 IP、录像编号或数量

#### Scenario: 配置变更写入审计

- **WHEN** 管理员在管理端保存或重载配置
- **THEN** 系统写入审计记录，包含动作类型、操作者、客户端 IP 及变更概要（如“保存 DVR 服务器列表”）

### Requirement: 审计数据保留 3 个月

系统 SHALL 仅保留最近 3 个月内的审计数据。超过 3 个月的记录 SHALL 被自动删除或从可查询结果中排除，且不可被管理员查询到。

#### Scenario: 查询仅返回 3 个月内数据

- **WHEN** 管理员请求审计日志列表（无论是否带时间筛选）
- **THEN** 仅返回 created_at 在最近 3 个月内的记录

#### Scenario: 过期数据被清理

- **WHEN** 系统执行审计数据清理（如定时任务或启动时）
- **THEN** 删除 created_at 早于 3 个月前的审计记录，或确保其不再被写入或保留

### Requirement: 审计查询仅管理员可见与可访问

审计查询入口与 API SHALL 仅对管理员可用。非管理员用户 SHALL 无法在 UI 上看到审计查询入口，且直接请求审计 API 时 SHALL 收到 403 禁止访问。

#### Scenario: 管理员可见审计入口

- **WHEN** 已登录用户角色为管理员
- **THEN** 管理端侧栏或导航中显示「审计查询」入口，并可打开审计查询页面

#### Scenario: 非管理员不可见审计入口

- **WHEN** 已登录用户角色非管理员
- **THEN** 侧栏或导航中不显示「审计查询」入口

#### Scenario: 审计 API 仅管理员可调用

- **WHEN** 已认证用户角色非管理员请求 GET /api/admin/audit
- **THEN** 系统返回 403，且不返回任何审计数据

#### Scenario: 未登录无法访问审计 API

- **WHEN** 未携带有效认证信息的请求访问 GET /api/admin/audit
- **THEN** 系统返回 401，且不返回任何审计数据

### Requirement: 审计查询接口与筛选

系统 SHALL 提供分页的审计日志查询接口，支持按时间范围、动作类型、用户名（可选）筛选，且仅返回当前用户有权限查看的 3 个月内数据。

#### Scenario: 分页查询审计列表

- **WHEN** 管理员请求审计列表并传入 page 与 page_size
- **THEN** 返回对应页的审计记录及分页信息（如总数、当前页）

#### Scenario: 按时间范围筛选

- **WHEN** 管理员请求审计列表并传入 from 与 to（时间范围）
- **THEN** 仅返回该时间范围内且仍在 3 个月保留期内的记录

#### Scenario: 按动作类型筛选

- **WHEN** 管理员请求审计列表并传入 action（如 login_success, play, config_save）
- **THEN** 仅返回该动作类型的记录
